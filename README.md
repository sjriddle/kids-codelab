# 🐢 Learn to Code, Kids! (ages 5–7)

A gentle, **highly visual** intro to Python for young children — and the grown-ups
learning alongside them. Everything lives in one friendly browser app, the
**Code Lab**: your child picks a lesson, reads a goal, then drags **knobs**
(sliders and color pickers that light up the matching line of code) or edits real
Python — and watches it draw, hears it play music, animates a scene, or turns it
into a little game.

Five tracks — 🎨 Draw, 🎵 Music, 🤖 Maze, 🏃 Move, 🕹️ Play — cover the foundations of
computer science (**sequence → loops → data → algorithms → events**), disguised as
play.

> **No installing anything.** Everything uses Python's built-in tools only, and
> all the editing happens in the browser — no separate text editor needed.

---

## 👋 Start here

1. Open **`SETUP.md`** and follow the 3 short steps (about two minutes).
2. Then just start the Code Lab:
   ```
   python3 code_lab/code_lab.py
   ```
3. Pick a lesson from the dropdown, read the **goal**, and play. Each lesson has
   a **💡 Grown-up tip** (tap to expand) with what to say and an unplugged warm-up.

**The golden rule for this age:** *you* drive the keyboard, *they* drive the
ideas — let them predict, choose, and shout "again!". Stop while it's still fun.

---

## 🎒 The five tracks

The Code Lab's lessons are grouped into tracks. Each one teaches a core idea of
computer science, disguised as play:

| Track | What you make | The big idea |
|-------|---------------|--------------|
| 🎨 **Draw** | Shapes, flowers, spirals | Instructions in order → **loops** → **variables & lists** |
| 🎵 **Music** | Tunes you can hear | Sequences, loops, and lists — but with **sound** |
| 🤖 **Maze** | Steer a robot to a star | **Algorithms & debugging** (fix the steps to reach the goal) |
| 🏃 **Move** | Little animations | **Things change over time** (the seed of state & frames) |
| 🕹️ **Play** | Games with keys & clicks | **Events** — the program reacts to *you* |

Together these cover the foundations: **sequence → loops → data → algorithms →
events.**

## 🧰 What's in the Code Lab

- **Lesson picker** — choose a lesson; **Next ▶** moves on; **↺** resets it.
- **🎛️ Knobs** — sliders and color pickers appear automatically for the variables
  in the code. Drag one and the matching **line of code lights up**, so your child
  connects the knob to the code. Great for kids who can't type yet.
- **📝 Code editor** — the real Python, with line numbers. Edit anything.
- **▶ Run** — or press **Enter**, or just move a knob. The result draws, plays,
  animates, or becomes a little game on the right, with a progress bar.

---

## 👩‍🏫 Tips for teaching ages 5–7

- **Keep sessions short** — 15–20 minutes. Two short sittings beat one long one.
- **Predict, then Run.** "What do you think will happen?" before every run.
- **Let them change one knob or number** and run again. Experimenting *is* the
  learning.
- **Do the unplugged warm-up** in the Grown-up tip — being "the robot" or
  "clapping a loop" teaches more than any screen.
- **Mistakes are great.** A red error message names the line to fix. Fixing it
  together is the lesson — that's real debugging.
- **Follow their joy.** If they love the music, write a longer song. If they love
  the maze, dream up a harder one.

### About `extras/`
The `extras/` folder holds the original numbered modules (drawing scripts, a
colorful Terminal/CLI set, a free-draw browser playground, and a choose-your-path
game). They've been **kept, not deleted** — they're a nice "read a whole real
program" resource for when your child is older and starting to read code, and
each still has its own `LESSON_PLAN.md`. You don't need them to use the Code Lab;
think of them as a bonus appendix.
